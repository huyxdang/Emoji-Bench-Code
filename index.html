Measuring a model's ability to detect and correct its own errors

Emoji-Bench evaluates metacognition: whether language models can naturally detect and correct their own errors. Each task presents a logical system in novel emoji-based notation with a target expression to derive. We inject an incorrect step into the prefilled model response and prompt the model to “Please continue.” Across eleven frontier models, the GPT-5.4 family performs strongest, naturally recovering from the error and arriving at the correct answer.

Dataset

Every problem is freshly synthesized at run time — no instance has appeared in any pre-training corpus, and the entire bench can be regenerated on demand. Two design choices make the resulting numbers worth trusting.

Choice · Symbols

Why emojis?

Emojis act as neutral, out-of-distribution operands. They tokenize as single units in modern tokenizers, carry no semantic prior that could leak the answer, and make every step trivial to verify by eye. A model can’t shortcut the derivation through memorized algebra or lexical priors when the alphabet is 🥟, 🪈, 🧊.

Method · Procedural

Procedurally generated, never seen.

Each formal system, operator table, and target expression is sampled at run time from a controllable difficulty grammar. No fixed problem appears twice, and nothing exists upstream to memorize. Difficulty scales by rule count, expression depth, and derivation length — producing the four buckets of 25 rows each (easy / medium / hard / expert).

Methodology

Emoji-Bench keeps the task coherent while removing the usual review cue. The model is handed an assistant transcript whose final step is wrong, asked only to Please continue., and scored on whether the continuation lands on the correct terminal symbol.

Define rules

The user message contains a procedurally generated emoji formal system, an expression to derive, and a strict step format.

Prefill a wrong step

The assistant transcript ends on a deliberately injected error. The model is told nothing is wrong.

Ask to continue

The reported setting uses exactly Please continue. — no review instruction, no inspection cue.

Score the terminal

The extracted Final Output: symbol must match ground truth, not the wrong branch implied by the prefilled error.

Analysis

The leaderboard scores the extracted terminal symbol. Behind that single bit, models differ qualitatively in how they handle the planted error. Quotes below are verbatim from predictions.jsonl.

Behavior · Self-detection

The model flags the prefilled error before fixing it.

Claude Opus 4.6 (reasoning max) frequently opens its continuation by naming the bad step and rolling the derivation back. This is the most legible failure mode — you can read it in the transcript.

“I notice I made an error in Step 2. Let me correct and continue…” claude-opus-4-6 · row easy-0006 · recovered ✓
Behavior · Silent recovery

The model corrects without saying so.

GPT-5.4 (reasoning xhigh) often continues straight from the wrong step but quietly substitutes the correct value, reaching ground truth without verbal acknowledgement. The headline metric counts this as a win; a self-detection metric would not.

“Step 2: ((…)) = (…) [by ⊕ table]” — terminal lands on the correct symbol; no flag, no apology. gpt-5.4 · representative row · recovered ✓
Behavior · Cascade

The model accepts the error and propagates.

Claude Opus 4.7 (reasoning max) frequently continues the wrong branch step-by-step, terminating on the planted-wrong final symbol. By construction, the cascade terminal is always distinct from ground truth, so this is a clean miss.

“Step 4: (🧊 ⊕ …) = 🧊  …  Final Output: 🧊” claude-opus-4-7 · row easy-0005 · cascaded ×
Behavior · Extraction failure

The model produces no parseable terminal.

Gemini 3 Flash (thinking high) generates partial work, malformed steps, or truncated output on most rows. The deterministic Final Output: regex fails to extract a symbol — counted as incorrect. A loose-judge metric would still likely rank it last, but this is the dominant cause of its 10%.

“Step 3: (🥟 ⊕ (🪵 ⊕ 🪈)) = (🥟 ⊕ 🪈)” — derivation truncates without a final terminal. gemini-3-flash-preview · representative row · extraction ∅
